# NSMusicS v2.3.1

## English

- This release is the current cross-platform transition release for the existing NSMusicS Electron and Web delivery line.
- Desktop packaging is prepared for Windows x64, Windows ia32, Windows arm64, Linux x64, Linux arm64, macOS x64, and macOS arm64.
- The current delivery focus remains local music libraries, private cloud music, compatibility with NineSong, Navidrome, Jellyfin, and Emby, plus the current Docker-oriented Web mirror line.
- This release should be treated as the bridge version before the next major refactor wave.
- The next refactored client wave is currently targeted for roughly mid-May 2026.
- That next wave is planned to include React, React Native, Vue, and Angular client lines.
- The next-generation Electron and Web clients are intended to follow the official and community-recommended best-practice direction of each stack instead of forcing a single shared implementation style.

## 中文

- 这是当前 NSMusicS 既有 Electron 与 Web 交付线的跨平台过渡版本。
- 当前桌面端打包已覆盖 Windows x64、Windows ia32、Windows arm64、Linux x64、Linux arm64、macOS x64、macOS arm64。
- 当前版本仍然聚焦本地音乐库、私有云音乐、NineSong / Navidrome / Jellyfin / Emby 兼容，以及现阶段的 Docker Web 镜像交付线。
- 这个版本应当被视为下一轮大重构正式落地前的桥接版本。
- 下一轮重构后的客户端矩阵，当前目标时间为 2026 年 5 月中旬左右。
- 下一轮计划包含 React、React Native、Vue、Angular 四条客户端线。
- 下一代 Electron / Web 客户端会分别遵循对应技术栈官方与社区推荐的最佳实现方向，而不是强行维持单一统一实现风格。
